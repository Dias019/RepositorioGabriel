Sistema de oficina

Explicação

O site que criei tem o intuito de funcionar como um sistema de oficina.

É necessário criar um login para acessá-lo.

Decidi criá-lo na web pela flexibilidade que o desenvolvimento web oferece, permitindo que qualquer pessoa acesse de onde estiver.

O site contém botões para diversos tipos de cadastros ou orçamentos.

Cadastro de clientes

Fornece as seguintes informações:

Nome do cliente e número de telefone.

Cadastro de funcionários

Fornece as seguintes informações:

Nome do funcionário, número de telefone, email e endereço.

Cadastro de fornecedores

Fornece as seguintes informações:

Endereço, número de telefone, nome do fornecedor e produto fornecido.

Cadastro de peças

Permite inserir o nome da peça, a referência e o valor.

Cadastro de Carros

Esse modal permite cadastrar os veículos dos clientes.

Você seleciona o cliente cadastrado e, em seguida, preenche os dados do veículo.

Por exemplo: Placa, Ano, Marca, Modelo, KM e Cor.

Criando orçamentos

Falando agora sobre orçamentos, foi desenvolvida essa página

que permite que o usuário preencha os campos fornecidos

Para a criação de seu orçamento.

Os primeiros campos pedidos, são os campos de cliente,

Como nome, e telefone.

Assim que preenchido com o nome (tendo a opção de selecionar o cliente),

O telefone já é puxado automaticamente.

Em seguida, temos a opção de selecionar o carro, após selecionarmos,

Os outros dados já são puxados automaticamente, como modelo, placa

Ano, KM, cor e marca, facilitando então, na criação de orçamentos.

Em seguida, nós temos os campos de peça, referência e valor da peça.

Sendo capaz de selecionar mais de uma peça, pois as vezes o serviço é grande

Após preencher os campos de peça, nós temos o campo de valor da mão de obra.

Preenchendo então o valor da mão de obra,

O sistema soma os valores das peças, junto com o valor da mão de obra, finalizando então

a aba de criação de orçamento, e dando sequência a próxima etapa.

Orçamento PDF

Após a criação do orçamento,

Temos um botão escrito "Criar orçamento",

Ao clicarmos em "Criar orçamento", o sistema pega os dados

dos campos de criação de orçamento, e gera um arquivo PDF.

Eu pensei em criar o arquivo PDF, para facilitar a impressão

do orçamento, e também por ser bem flexível quanto ao

encaminhamento, facilitando enviar o mesmo seja por WhatsApp,

Email, Telegram e etc, pois o arquivo será baixado em

sua máquina, selecionando a pasta desejada.

Uma dica que eu dou aos usuários, é que criem uma pasta de orçamentos

e dentro, crie uma pasta com o ano, e dentro dessa pasta do ano,

crie uma pasta com os meses, para salvar os arquivos separadamente,

facilitando a consulta de orçamentos quando necessário.

Lembrando que o arquivo PDF vem com a Logo da empresa, os dados

do cliente e de seu veículo.

E no fim da página, uma barra para o cliente assinar, para saber

se o orçamento foi aprovado, com data e hora.

Ferramentas e Dificuldades

Para armazenar os dados do nosso sistema, tanto de

Login, quanto do próprio sistema de oficina, eu escolhi

usar o MySQL, no Workbench. Por ter mais intimidade com o

programa e mais conhecimento da ferramenta que é o MySQL,

tanto que não obtive nenhuma ajuda ou auxílio para criar o BD.

Para criar o Backend do sistema, eu optei por Node.js e JavaScript, mesmo

não tendo muita intimidade com a ferramenta, acredito que seja a

melhor para esse tipo de serviço, então pedi ajuda ao ChatGPT,

para criar o servidor do meu sistema e "linkar" com meu Banco de Dados.

Tive muita dificuldade, pois não tenho conhecimento da linguagem, mas com

muita pesquisa e força de vontade, fiz acontecer.

Para criar a lógica do meu relógio na tela inicial, usei o JavaScript.

Já para a interface visual (frontend), eu usei o HTML e o CSS

por ter mais familiaridade com as ferramentas, como já comentado,

eu queria criar o sistema como um sistema WEB, por acreditar que a flexibilidade

seja maior, sendo possível acessar de qualquer lugar e dispositivo, sem precisar

baixar nada.

Minha maior dificuldade foi com o Backend, como já mencionei,

mas não tinha familiaridade nenhuma

com o Node.js nem o JavaScript. Mas isso não me impediu de criar.

Tive essa ideia de criar o sistema, por já ter trabalhado em uma oficina mecânica.

Juntei a vontade de criar um sistema com a minha realidade cotidiana, e então criei esse projeto.

O programa que usávamos na oficina era um app que era transferido via pen drive, e um dia esse pen drive

acabou quebrando, e perdemos o sistema. Meu chefe teve que entrar em contato com o dono do app para adquirir

outro sistema, e acabamos perdendo alguns dados salvos, por isso optei por baixar os arquivos em PDF.
